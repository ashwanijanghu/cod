COD

THE UNASSUMING DOC FORMAT

/**
@Example
  Text can go anywhere.

  @flag
  @number 42
  @string Hello, cod
  @nested
    Nested text.
    @property yay
  @list A
  @list B
  @list C
*/

{
  "Example": {
    "!text": "Text can go anywhere.\n",

    "flag": true,
    "number": 42,
    "string": "Hello, cod",
    "nested": {
      "!text": "Nested text.",
      "property": "yay"
    },
    "list": ["A", "B", "C"]
  }
}

Cod is a language-agnostic documentation format that produces simple, nearly "line-for-line" JSON.

That's all it does.

The flexible nature of Cod affords its users a great deal of control:

  • Integrate Cod's output into your existing web templates.
  • Use any markup language in text sections (or none at all).
  • Create domain-specific @tags for concise clarity.
  • Use a single doc format across multiple languages.
  • Easily compare specific versions of your API using diff.

Cod isn't for everybody. If you just need quick, human-readable docs, use an all-in-one solution like Doxygen or Sphinx.

See its GitHub page for details.
